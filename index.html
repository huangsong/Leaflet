Leaflet 是一个对移动设备友好的开源交互地图开发 JavaScript 框架。 轻量级，Gzip压缩后仅 33 KB, 它拥有大多数地图开发者需要的 功能 。

Leaflet 以 简洁, 高性能 和 易用性 为理念。 在移动端和桌面端都能高效的运行,并且具有可扩展性和很多的 插件， 还有简单易用的 API文档 和简单的、可读性强的 源代码。

现在我们在一个 'map' div 标签中创建一个地图容器, 并添加一个 自定义的瓦片图层，然后在图层上面添加一个带文字信息的冒泡窗的图标标记:

var map = L.map('map').setView([51.505, -0.09], 13);

L.tileLayer('http://{s}.tile.osm.org/{z}/{x}/{y}.png', {
    attribution: '&copy; <a href="http://osm.org/copyright">OpenStreetMap</a> contributors'
}).addTo(map);

L.marker([51.5, -0.09]).addTo(map)
    .bindPopup('一个漂亮的 CSS3 冒泡窗.<br> 易于定制。')
    .openPopup();

通过 快速开始指南, 和 其他示例了解更多内容， 或直接阅读 API 文档. 如果你有任何疑问，请访问 帮助 。

功能

Leaflet 并不想做一个大而全的框架，相反更专注于把基本的功能做到完美。

图层

  • 瓦片图层, WMS
  • 标记, 冒泡窗
  • 矢量图层: 折线, 多边形, 圆, 矩形
  • 图片叠加层
  • GeoJSON

交互功能

  • 拖动惯性平移
  • 滚轮缩放
  • 手势缩放
  • 双击放大
  • 拉框缩放 (按住 Shift 键拖动)
  • 方向键平移
  • 事件: 点击, 鼠标悬停等等。
  • 标记拖动

视觉特效

  • 缩放平移动画
  • 瓦片渐变动画显示
  • 非常漂亮的默认设计： 标记, 冒泡窗和地图控件
  • 支持视网膜分辨率

自定义功能

  • 纯 CSS3 冒泡窗和地图控件 便于修改
  • 图片 - 基于 HTML-based 的标记
  • 一套创建自定义图层和控件的 简单接口
  • 自定义地图投影 ( EPSG:3857/4326/3395)
  • 强大的 OOP 设计模式， 可快速扩展现有的类

性能特点

  • 在移动端使用硬件加速 感觉如原生应用般
  • 利用 CSS3 特性 让平移和缩放更加平滑
  • 智能折线、多边形渲染 动态裁剪和简化
  • 模块化构建系统 按需加载
  • 在移动端消除点击延迟的问题

地图控件

  • 缩放按钮
  • 版权信息
  • 图层切换
  • 比例尺

浏览器支持

桌面端

  • Chrome
  • Firefox
  • Safari 5+
  • Opera 12+
  • IE 7–11

移动端

  • Safari for iOS 7+
  • Android browser 2.2+, 3.1+, 4+
  • Chrome for mobile
  • Firefox for mobile
  • IE10+ for Win8 devices

杂项

  • 轻量级
  • 没有外部依赖

If you find some feature really missing in Leaflet, first check if there's a plugin for it. If not, please vote for the feature on the Leaflet UserVoice page.

Getting Involved

Let's create the best mapping library that will ever exist! Leaflet is developed by Vladimir Agafonkin of Mapbox with a team of dedicated contributors. Pull requests are always welcome. However, there are many more ways to get involved with the development of Leaflet.

You can help the project tremendously by discovering and reporting bugs, improving documentation, helping others on the Leaflet forum and GitHub issues, showing your support for your favorite feature suggestions on Leaflet UserVoice page, tweeting to @LeafletJS and spreading the word about Leaflet among your colleagues and friends.

Check out the contribution guide for more information on getting involved with Leaflet development.
